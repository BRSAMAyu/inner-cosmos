# Inner Cosmos 完全体目标模式总入口

> 状态：ACTIVE / AUTHORITATIVE
> 目标类型：长期自主实现目标，而非单轮工作包
> 最后对齐：2026-07-20
> 适用对象：接管本仓库并以 Goal Mode 持续工作的 Codex/Agent 团队

## 1. 唯一总目标

将当前 Inner Cosmos 原型持续演化为一套可真实使用、可在新加坡部署运营、以 Kubernetes 为运行底座、同时具有鲜明 AI 创新与卓越体验的完整产品：

> **一个拥有时间感、主动性、连续自我与关系演化能力的 Aurora，帮助用户把自然表达转化为长期自我理解；再以高保真、可授权的共鸣体和慢社交机制，把这种理解转化为人与人之间更低压力、更有质量的真实连接。**

本目标不是“完成一次重构”“迁移到 EKS”或“把测试跑绿”。只有产品、AI、数据、体验、云原生、移动端、安全合规和真实部署能力共同达到最终验收规约，目标才算完成。

## 3. 结果契约

最终系统必须同时具备：

- 可令人眼前一亮的 Aurora：多消息、停顿、思考、打断、停止、重规划、时间感、主动唤醒、长期记忆、连续自我、关系与人格演化。
- 不依赖“一个超长 Prompt”的 AI runtime：双核推理、编排器、记忆与画像检索、工具/Skill、策略与 critic、结构化状态、评测和回放。
- 高精度用户模型：事实、偏好、关系、价值、情绪、习惯、语言风格、需求、目标、边界、置信度、来源、时效和用户纠正闭环。
- 高保真共鸣体：从真实授权 trajectory 编译动态人格，能保持事实、风格、习惯、边界与时序一致性，并支持模拟器和多策略匹配。
- 有产品价值的心理学能力：以可审计 Skill 形式提供自我反思、量表或模型辅助，但不冒充诊断、治疗或专业人员。
- 美观、连贯、有情绪质感的跨端体验：流动、呼吸、星尘、涟漪和半透明构成统一视觉语言；白天温暖、夜晚深邃；移动端是一等公民。
- 面向真实生产的工程系统：Java 21、Spring Boot 3.5、PostgreSQL/pgvector、Redis、异步事件、React/TypeScript PWA、Capacitor、EKS、IaC、可观测、灾备和安全供应链。
- 面向新加坡发布的产品能力：英文优先可用、中英国际化、时区与本地资源、PDPA 数据权利、跨境数据治理、商店和运营准备。
- 面向课程与商业的双轨部署：`local-complete` 保证完整产品效果，`academy-eks` 证明受限教学环境中的 Kubernetes 能力，`commercial-sg` 保留真实新加坡生产架构；不得把三者的证据混为一谈。

上述创新能力不得因“更容易实现”被删除、静态化或改成普通聊天壳。安全与治理应通过授权、分层、可观测、可撤销、评测和 fail-safe 实现，而不是用无意义的体验阉割代替工程治理。

## 4. 自主执行原则

- 从仓库和证据获取事实，自己拆解阶段、建立依赖、实施、测试、复核和修复；不要等待人类逐条分配工作。
- 可在安全范围内自主创建工作包、ADR、迁移脚本、测试、评测集、基础设施和文档；持续维护机器可读验收账本。
- 优先形成可工作的纵向闭环，随后扩展深度；不允许只搭空壳、只写 YAML、只写文档或只堆页面。
- 以 `16` 定义的产品战役作为主要执行单元；旧界面、旧 API 和原型技术选型没有天然兼容权。允许为显著更好的最终体验重构或替换，但必须保护用户数据、安全边界和可复现证据。
- 至少一半有效研发投入直接服务用户可感知的体验或 AI 效果；不得用持续增加平台、类和测试数量回避 Aurora、记忆画像、共鸣体与 UIUX 的质量难题。
- 对每个关键 AI/产品能力建立离线评测、场景回放和真实体验验收；“代码存在”不等于“效果成立”。
- 每一阶段保持主干可构建、关键旅程可演示、数据可迁移、失败可回滚。
- 保留无关工作树改动；先检查分支、状态和已有证据，避免覆盖其他 Agent 的工作。
- 优先自主解决不确定性：查源码、查现有文档、运行实验、写小型 PoC、对比方案。只有真实的人类权限或不可逆外部决策才升级询问。
- 采用单会话持续 Goal 时，每个提交、战役检查点和上下文压缩都只是恢复点；按 `17` 维护持久化执行状态并自动进入下一差距，不以“本轮已完成”结束顶层任务。

## 5. 不可逾越的人类门禁

下列事项必须由有权限的人类完成或明确授权，Agent 不得伪造证据：

- 外部 Provider 密钥吊销、轮换及独立签字。
- AWS、DNS、商店、支付、法务主体等真实账户中的不可逆操作。
- 向真实用户开放、生产数据迁移、跨境数据处理和正式隐私/法律文本批准。
- 真实心理健康专家审阅、真实用户研究同意和高风险研究伦理决定。
- 发生可能破坏现有数据或公开系统的操作。

门禁阻塞某条路径时，记录阻塞证据并继续所有不依赖该门禁的工作；不得把单个外部门禁当作整个目标停止的理由。

## 6. 完成定义

只有满足以下条件才可宣布 COMPLETE：

1. `docs/goal/complete-product-acceptance.yml` 中所有必需验收项均为 `PASS`，且引用可复现证据。
2. `对齐文档/12-完全体实现路线与最终验收规约.md` 的 G0—G9 全部关闭。
3. 核心旅程在 Web、PWA 和移动容器中通过真实端到端验证。
4. Aurora、记忆画像、共鸣体、心理 Skill 和匹配均通过质量评测与人工体验门槛，而非仅有 Mock 流程。
5. EKS 环境完成部署、扩缩容、故障恢复、观测、备份恢复和安全验证。
6. 新加坡发布所需的合规与运营门禁已由授权人员签字；若项目范围明确只到发布候选，则必须标记 `RELEASE_CANDIDATE_BLOCKED_BY_HUMAN_GATE`，不得冒充已公开商用。
