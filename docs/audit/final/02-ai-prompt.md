# Expert 2 — AI / Aurora / Prompt Pipeline

> Senior authority synthesis over three sub-aspect inspections (prompt engineering; agent & reply-strategy logic; emotion modeling, memory extraction & self-understanding). Overlapping findings deduped, severity re-rated against real production impact, related items merged, and gaps the subagents missed added after direct re-read of the load-bearing files.

---

## Executive Summary

**What's strong.** This is a thoughtful, safety-first pipeline. The reply path runs `SafetyService.check` *before* any LLM token, crisis never streams as free-form consolation, and there is a genuine three-layer fallback (retry+backoff → JSON-repair re-prompt → deterministic mode/state-aware fallback) so a user always gets an in-character reply even when every provider is down. `PromptBuilder` is a disciplined single chokepoint with explicit curation caps (PORTRAIT_CONFIDENCE_THRESHOLD=0.45, PORTRAIT_MAX_DIMS=5, CORRECTION_MAX=5, per-field char caps) and every user-derived signal is sanitized. The IC-EMO three-phase emotion design is well-conceived and largely implemented (LLM + deterministic lexicon fallback, single EmotionWeatherMapper source of truth, EWMA baseline with anti-thrash confidence cap at 0.85). The two-timescale separation is correct in intent: MomentMood feeds the prompt while only the nightly baseline may move portrait dims. The in-reply anti-repetition machinery (bigram Jaccard + leading-clause detection against recent Aurora messages and within-reply, plus `[[SILENCE]]` semantics) directly addresses the "Aurora sounds repetitive" failure mode.

**What's broken (headline risks).** Three of the product's signature promises are silently no-ops in production:

1. **"Aurora re-learns who you are every 5 turns" is a lie.** The 5-turn reflection hook (`AuroraAgentServiceImpl.java:283`) calls the LLM and throws the result away — portrait only ever updates on goodbye-close or nightly. The portrait the user sees can be stale for an entire long session. This directly defeats RUN-006's "画像可视化 + 校准" premise. (P0)
2. **Mode switching edits prompt *text* but never changes sampling behavior.** `LlmRequest` has no temperature field; every provider hardcodes 0.72; the whole `ModeStrategy.temperature()` axis is inert. SLEEP_REVIEW (meant to be 0.6) and DAILY_TALK (0.85) run identically. (P1)
3. **The carefully engineered system boundary is sent as a `role:user` message**, while every provider injects its own one-line English `role:system` persona. Safety/anti-diagnosis/anti-injection rules sit in the user slot competing with user content. (P1)

Plus two data-integrity time-bombs: **gravity time-decay is dead** (`daysSinceLastTouched` is always 0 — the starfield is frozen at creation-time ordering), and **EmotionTimeline is never auto-aggregated** (the whole weekly-review / pattern / spectrum layer runs on empty data unless an admin hits a manual endpoint).

**Secondary risks.** Proactive pushes to offline users are silently dropped but still logged as sent, burning the daily budget with zero delivery (the "Aurora reaches out when she notices a mood drop" feature fails the exact users it targets); the entire 11-strategy `AgentReplyStrategy` hierarchy is dead code; the prompt-versioning subsystem (DB + admin UI + rollback + metrics) is a finished feature nothing reads; the hard-boundary refusal is gated on an optional service being non-null, so disabling continuity silently disables identity-violation protection.

## Strengths

- **Safety-first ordering.** Both `replyRich` and `stream` invoke `SafetyService.check` synchronously before any LLM token; crisis paths never stream free-form consolation (`AuroraAgentServiceImpl.java:150-156, 329-351`). Correct priority for a mental-health-adjacent product.
- **Layered, coherent fallback.** `callWithRetry` (retryable detection + exponential backoff) → `StructuredAiService` JSON-repair retry → deterministic `fallbackAuroraResult`/`fallbackSegments` that are mode- and state-aware → user always gets a coherent reply.
- **Disciplined PromptBuilder chokepoint** with explicit curation caps preventing prompt bloat (`PromptBuilder.java:17-36`).
- **Injection-conscious design**: `sanitize()` collapses whitespace (defeats role/line forgery) and strips common injection verbs, applied at every user-derived metadata entry (portrait, corrections, calibrations, momentEmotion, stateSignal, baseline).
- **Correct non-recitation restraint wrappers** on every perceptual block (画像/关系/此刻情绪/基线/更正), matching the vision's anti-medicalization stance.
- **Genuinely robust `StructuredOutputParser`** for common model pathologies: `<think>`/`<analysis>` stripping, markdown-fence extraction, brace-matching extraction, single-object-in-array unwrap, and a clever `escapeBareQuotesInsideStrings` heuristic for reasoning models emitting raw quotes in Chinese strings.
- **IC-EMO design well-implemented**: EmotionInsightService (LLM + deterministic lexicon fallback), EmotionWeatherMapper single source of truth, EmotionSpectrumDeriver deterministic, EmotionBaselineService EWMA with confidence capped at 0.85.
- **Two-timescale separation enforced in intent**: MomentMood (jittery) feeds 此刻情绪; only baseline moves portrait dims, and only nightly.
- **Honest telemetry surfaced to UI** (aiState, providerPolicy) and `ABTestService` metrics in `StructuredAiService` finally-block give real observability into which model answered and whether it succeeded.
- **Mature concurrency handling**: `EmotionInsightServiceImpl.insertWithRaceFallback` handles the duplicate-insert race via the `(user_id, source_session_id)` UNIQUE guard instead of leaking `DuplicateKeyException`.

## Vision Gaps — what this area SHOULD become

- **A true agent loop with explicit tool/ability calls.** Today "Aurora" is a single structured-LLM-call with post-hoc cleanup. The vision names an agent loop (`agentLoopPolicy`, `speakCount`, `continueReason`) but there is no multi-step reasoning — no planner, no ability to pull a specific memory by id, no ability to decide "I should check the emotion timeline before replying". A real loop (observe portrait+state → decide intent → optionally call memory/emotion tools → draft → self-critique against anti-repetition → emit) would materially raise reply quality and make the "how Aurora decides" story honest.

- **A closed, durable self-understanding feedback loop.** Recording a correction (or a 5-turn reflection — ai-001) should durably reshape Aurora's model with a `source` badge (dialogue-inferred / baseline-bridged / user-corrected) so the portrait page can show "you told me this" vs "I noticed this", Aurora can weigh them accordingly, and `reflectOnTurn` treats user-sourced dims as hard priors. This is the literal meaning of the branch name `feat/run006-aurora-self-understanding` and the pipeline does not yet close it.

- **A single unified outgoing-message pipeline.** Every Aurora-originated message — proactive, greeting, mode-ack, chat reply, capsule/persona chat — should flow through one pipeline applying the same safety check, anti-repetition, boundary sanitize, and persona consistency. Today the proactive path bypasses SafetyService entirely (ai-013), capsule/persona routing is a third parallel abstraction, and consistency is by convention.

- **A unified reply-routing layer.** The product has three parallel, inconsistent routing abstractions — `AgentReplyStrategy` (dead, ai-008), `ModeStrategy` (alive but temperature-inert, ai-002), and the implicit persona/capsule paths. There should be ONE strategy registry that chat/capsule-chat/persona-chat/thought-shredder all dispatch through, so mode/persona/safety selection is centralized, testable, observable — and mode actually changes sampling parameters.

- **Live prompt versioning wired for real.** The fully-built versioning subsystem (DB + admin UI + rollback + metrics + A/B — ai-005) should resolve boundary/mode/schema segments from `PromptVersionService` with code as seed/fallback, so prompt edits ship without redeploy and the existing admin UI becomes real. Today it is an orphan that misleads.

- **Grounding/RAG for safety resources and authorized facts.** Crisis referrals should be accurate rather than model-hallucinated (ai-027), and cited memories should carry their actual text not just IDs.

- **A fresh, aging starfield.** Gravity should age (ai-004) and re-touching a memory should bump it, so the starfield surfaces recent relevant memories instead of a frozen creation-time ordering. This is the product's signature visualization and it is currently static.

- **An always-fresh emotion-understanding layer.** EmotionTimeline should auto-aggregate from traces on the DialogFinished/nightly path (ai-005) so weekly-review / pattern / spectrum are always fresh — not fictional. Consolidate to one weekly-review service with a persisted V2 schema and a unified 0-10 scale (ai-022, ai-026).

- **Determinism + replayability for quality regression.** `LlmRequest` should carry temperature/topP/seed (ai-002) plus a deterministic mode pinning the mock client for replayable reply golden-tests, so voice/safety/robustness regressions are caught end-to-end, not just by unit string-presence checks. Include adversarial golden cases (injection, crisis, mid-JSON truncation, multi-message-disabled).

- **Self-understanding provenance + auditability.** Every portrait dim should carry source + confidence (ai-036); extend the existing `gravityExplanation` pattern (MemoryServiceImpl.java:335) to portrait dims so "why Aurora understands this" is auditable — which dialogues/themes/baselines produced this belief — not a black box.

- **Editable self-understanding UX.** A "我的更正 / Aurora 对我的理解" page where the user can review, amend, and retire past corrections (ai-037), closing the "这不太是我" loop visibly instead of leaving it write-only.
