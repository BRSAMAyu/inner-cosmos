# Expert 3 — Frontend, UI/UX & Experience

> Final synthesized report. Deduplicates, re-grades, and merges 4 subagent passes; every P0/P1 claim below was re-verified by reading the source directly (file:line cited).

---

## Executive Summary

**What's strong.** The Inner Cosmos frontend is genuinely art-directed, not templated. There is a consistent Morandi palette with a complete, internally-consistent dark ("星空") theme (every surface/accent/text/line/shadow token re-derived, no navy leakage), an aggressive global reduced-motion contract (CSS zeroes all animation **and** `motion.js` tears down RAF loops), real accessibility primitives (skip-link, `:focus-visible` halo, focus-move-in + focus-return modals, `aria-live` on the loading mask, `lang=zh-CN` everywhere), strong XSS hygiene via a 5-entity `IC.esc()` applied even to streamed SSE chunks, and resilient SSE on the core Aurora loop (explicit POST fallback on pre-token failure). RUN-003 (capsule resonance surfacing) and RUN-005 (the "这不太是我" correction loop) are genuinely novel, well-built UX moments. The `portrait.html` calibration overlay is the a11y high-water mark for the rest of the app to follow.

**What's broken (headline risks).** Beneath the polish there are **correctness and trust defects** that any reviewer or first-time user will hit within minutes:

1. **The product has no real front door.** Every core page silently auto-logs the visitor into a *shared* demo account (`IC.ensureDemoLogin()` → `POST /api/auth/login demo/demo123`) on first contact. For a privacy-first, P0-owner-only self-reflection product, a brand-new visitor instantly impersonates the demo user and reads demo's P1 memories, letters, and starfield. This is simultaneously a UX-clarity failure and a data-exposure risk. (P1)
2. **`IC.api()` is status-code blind.** It calls `res.json()` unconditionally; a 401/403/HTML error page throws into a generic catch that toasts "连接暂时没有回应". There is **no 401/expiry handler anywhere** — sessions that expire mid-conversation strand the user with cryptic network errors and never redirect to login. (P1)
3. **Retries fire on ALL methods including non-idempotent POSTs.** A transient timeout re-sends `createSession`, `letterSend`, `createCapsule`, `todoCreate`, `letterReplyWithLetter`, `personaChatMessage`, `shredderProcess`, `deleteAccount` — risking duplicate letters/sessions/capsules. (P1)
4. **A "仅看一次" privacy promise is silently violated.** thought-shredder `DISPLAY_ONCE` mode persists the distilled card (`status=TRANSIENT`) *and* raw vent excerpts into `ThoughtFragment.rawExcerpt`, while `history()` filters only by `memory_type=SHREDDER` with **no status filter** — so the text the user expected to evaporate remains retrievable and contributes to gravity/starfield. (P1)
5. **Three reflection pages are completely orphaned** (zero inbound links): `thought-shredder.html`, `ai-log.html`, `ai-dev-history.html`. The shredder is one of the three core reflection tools; ai-log is a stated product value (transparent AI calls). They are unreachable without typing the URL. (P1)
6. **Broken social journeys:** "写慢信" from the friends list passes `?to=` which `slow-letter.html` never reads (always lands on a "no receiver" wall); reply-with-letter saves as `DRAFT` while the UI tells the user the letter "已封缄…在对方安静的时刻抵达"; the inbox reading-ceremony envelope auto-fires so the "touch to open" affordance never happens. (P1)
7. **Intended typography only loads on 1 of 35 pages.** The calligraphic identity (Noto Serif SC / Cormorant / LXGW WenKai) is fetched only by `index.html`; the other 34 pages fall back to generic serif/cursive. The signature visual identity is absent everywhere except the landing page. (P1)

**The deeper pattern.** This is a 35-page vanilla-HTML app with no build step and no shared head include, so every page re-declares its own scripts, re-mounts the ambient stack, and drifts. Design tokens are declared but dead (`--fs-*` consumed 0 times while `h1` font-size is hardcoded across 9 media queries; `--ring` consumed 0 times; a third `color-scheme: light` can override the JS dark theme). There are **three divergent modal implementations**, three near-duplicate cursor CSS blocks, a shipped-but-unreachable styleguide (`ui-preview.html`), an un-cache-busted `app.css` (stale CSS after deploys while `app.js` busts), and a 22-item nav with no mobile pattern. The frontend's ceiling is gated by missing *infrastructure* (a shared shell/router, an enforced token system, a single modal component, a mobile nav), not by per-page effort.

No single page is catastrophic; the aggregate is "beautiful surface, soft foundations."

## Strengths

- **Dark theme is thorough, not bolted on.** `app.css:1681-1800` re-derives every token plus per-component overrides (panel/card/bubble/input/modal/toast/skeleton) — a complete palette.
- **Reduced-motion is global and aggressive.** `app.css:2636-2644` zeroes all animation/transition/scroll-behavior; `motion.js` tears down RAF loops and injected cursor DOM. Not a token CSS toggle.
- **Accessibility primitives exist and are mostly correct.** Skip-link injected as first focusable (`app.js:217-232`), `:focus-visible` halo, 44px `--tap`, modal focus-move-in + focus-return-to-trigger (`app.js:387-431`), `aria-live` on loading mask, `lang=zh-CN` + viewport on every page.
- **XSS-hardened, not naive.** `IC.esc()` (`app.js:243-250`) is a proper 5-entity escaper applied consistently before `innerHTML` — including on streamed SSE chunks (`aurora-chat.html:717`) and starfield drawer content. Reading ceremony splits *already-escaped* text; no raw body injection path.
- **Resilient core loop.** Real SSE streaming with explicit POST fallback on any pre-token failure (`aurora-chat.html:991-1028`), an 8s proactive timeout, and an explicit send-lock (`auroraSending`) to prevent double-send.
- **Reading-ceremony reduced-motion contract is exemplary** (`inbox.html:175-220, 516-604`): escaped body always in the DOM, sealed state is the *only* thing that hides it, 7s hard safety timeout, try/catch fail-open. The right way to animate over content.
- **RUN-003 resonance surfacing** (matchScore + ≤2 reason chips, restrained) and **RUN-005 "这不太是我" correction** (hover affordance + modal quoting the line + success micro-feedback + marked bubbles) are genuinely novel, on-brand UX.
- **`portrait.html` calibration overlay** is the a11y model: `role=dialog`/`aria-modal`, Escape-to-close, focus management, defensive JSON parsing, confidence-bar animation, graceful empty state.
- **Admin destructive actions** all require a reason string and write to audit — matches the stated "技术而温暖" principle.
- **Consistent use of `translate`** (not `transform`) for hover lifts so they compose past `card-reveal`/`thread-reveal` entry animations whose `forwards` pins `transform`.

## Vision Gaps — what this area SHOULD become

1. **A real first-run onboarding journey (no front door today).** Brand-new users are silently impersonated into the shared demo's populated cosmos (FE-01). The product needs an intentional *landing → sign-up → empty personal cosmos* flow, plus a guided 3-step tour ("说一句今天最真实的话" → "看它变成一颗星" → "决定要不要分享"), and a privacy moment that teaches the actual P0→P3 data-layer promise ("你的原话只属于你；只有被抽象的回声可以被分享") — the landing mentions "安全边界" abstractly but never teaches the differentiator.

2. **A shared app shell / SPA-lite router + enforced design tokens.** 35 HTML files each re-declaring scripts, re-mounting the ambient stack, and drifting is the root cause of FE-10/FE-29/FE-48/FE-49/FE-53/FE-54. A minimal client router (or shared head include) would unify font loading, cache-busting, script includes, and `mountShell` once — and let the dead `--fs-*`/`--ring` tokens finally be adopted so "artist-level polish" has one source of truth and a reachable living styleguide.

3. **Centralized, typed request handling.** `IC.api` needs status-code awareness (FE-02), method-aware retry (FE-03), an auth-expiry redirect interceptor, and a standard error-result shape — today every page re-implements `if (!r.success) toast(...)` inline (~30+ times in `dashboard.html` alone). Pair with a shared empty/loading/error component contract (`<ic-empty>`, `<ic-error>`, hideLoading-with-retry).

4. **A unified "connection home" merging inbox + threads + friend requests.** Today these are three siloed pages (`inbox.html`, `letter-threads.html`, `social.html`) with duplicated empty-states, no cross-linking except one buried link, no live notification dead-angle coverage (FE-21), orphaned drafts with no "续写/投递" entry (FE-07), and a one-directional resonance loop with no "this landed for me" feedback. A single slow-relationship graph home with live badges, in-place reply, and draft续写 would realize the "慢社交" promise.

5. **A "reflection hub" / journaling home.** heart-diary, thought-shredder, daily-record, weekly-review, and portrait are scattered — and thought-shredder is unreachable (FE-05). Unify them into one guided entry flow, add an AI-prompted reflection nudge ("Aurora gently asks…"), let users set memory-importance from wherever they encounter an insight (not only the star map), and add an exportable/shareable reflection artifact (PDF or sanitized P2 echo) — a natural "wow" moment for a self-resonance product.

6. **Proactive safety routing.** The harbor exists but nothing detects rising distress from emotion-timeline/shredder to surface `safety-harbor.html`; the proactive-safety path is one-way (user must find it). Wire a mood-check-in that can route users into the harbor, and fix the harbor's a11y (FE-28) since it's the highest-stakes page.

7. **A capsule "window-shopping" surface and persona-chat session index.** The plaza modal shows intro/tags/boundary but the only way to hear a capsule's voice is to start a full chat (consuming a daily round) — a zero-cost pinned first utterance would let browsers explore without spending quota. Sessions are also orphaned in sessionStorage with no "my ongoing echo conversations" resume surface (FE-18).

8. **A mobile navigation pattern and cross-device responsiveness.** With 22 nav destinations the current hidden-horizontal-scroll + wrap-into-a-wall approaches both break (FE-15, FE-04 in the infra pass); the responsive starfield (FE-15) and mobile nav are core missing pieces for the stated "cross-device perfection" ambition.

9. **An honest "Aurora availability" model and unified AI-state UI.** Replace the duplicated, partly-dead, unconditionally-pulsing AI-status surfaces (FE-44) with one chip (green=real model, amber=Mock/fallback) shown consistently — important because the product's trust contract depends on the user knowing whether they're talking to a real model or a fallback.

10. **A closed safety/feedback loop.** `reportLetter` fires and toasts "举报已提交" but gives no status back to the reporter and no "reported" badge; mute/block/report lack closure. Close the loop with reporter-visible status and card badges.
