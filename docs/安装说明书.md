# Inner Cosmos 安装说明书

## 环境要求

- **JDK 17 或更高版本**（项目以 Java 17 为编译目标，已在 JDK 17 与 JDK 21 上验证可运行）。
- 仅"从源码运行"时需要 Maven 3.8+；若本机没有 Maven，可使用 `scripts/run-dev.ps1` 自动下载本地 Maven，或直接用项目自带的 `./mvnw`。
- **默认演示无需 MySQL，也无需配置任何 API Key 或环境变量**——系统使用 H2 文件数据库（落盘到 `./data/`），并**默认即接入真实大模型 GLM glm-4.7**（真实 key 已内置在提交的 `src/main/resources/application.yml`，私有仓库，开箱即真实 AI）。若网络/额度异常会自动回退 Mock，演示不中断。
- 天气（open-meteo）与时间（浏览器本地）本就是真实数据，无需任何 key。
- 如需 MySQL，请安装 MySQL 8 并创建 `inner_cosmos` 数据库（见下方"MySQL 运行"）。

## 方式一：运行已打包 JAR（推荐——老师无需安装 Maven）

提交包内已提供可执行 JAR `inner-cosmos-0.1.0.jar` 与启动脚本 `run-jar.bat`（位于同一目录）。

- **Windows：** 双击 `run-jar.bat` 即可（脚本会自动定位同目录下的 JAR 并启动）。
- **任意系统命令行：** 在 JAR 所在目录执行：

```bash
java -jar inner-cosmos-0.1.0.jar
```

首次启动约需 10–20 秒（初始化 H2 数据库、建表与种子数据）。启动后访问：

```text
http://localhost:8080/pages/index.html
```

默认账号：

```text
admin / admin123
demo / demo123
```

> **注意：** 默认即接入真实大模型 **GLM glm-4.7**（key 已内置，无需配置任何 API Key 或环境变量），开箱即真实 AI——对话、主动问候、记忆沉淀、情感重力、共鸣体、慢信件、10 维画像、Aurora 自我模型等全部可见。若网络/额度异常自动回退 Mock，演示不中断。如需切换供应商或自带 key，见下方"远程模型配置"。

创建数据库：

```sql

```

修改 `src/main/resources/application-mysql.yml` 中的账号密码，然后运行：

```powershell
.\scripts\run-dev.ps1 -Profile mysql
```

## 远程模型配置

**默认即真实大模型，无需任何配置。** 提交的 `application.yml` 默认值如下（key 已内置，老师开箱即真实 AI）：

```yaml
llm:
  mode: prod            # 默认生产模式，走真实模型
  provider: glm         # 主供应商：智谱 GLM
  failover-providers: glm,minimax   # 失败转移链 GLM → MiniMax，最终 Mock 兜底
  glm:
    model: glm-4.7      # 智谱旗舰，请求里已关闭 deep-thinking 以保证约 6–8 秒响应
  minimax:
    model: MiniMax-Text-01   # 失败转移备用模型（干净输出）
  asr-provider: mimo    # 语音转文字：MiMo-V2.5-ASR（key 已内置，开箱即真实语音识别）
  mimo:
    asr-model: mimo-v2.5-asr
```

> **语音转文字（ASR）**：Aurora 对话框支持语音输入，后台用 **MiMo-V2.5-ASR**（key 已内置）做真实转写，支持中英双语与自动语种检测；转写失败自动回退占位文本，不阻断对话。
> **降级备份电路**：聊天链路为 **GLM glm-4.7 →（失败）MiniMax-Text-01 →（再失败）Mock 兜底**；语音链路为 **MiMo-ASR →（失败）Mock**。任一模型不可用都会自动切到下一个可用项，用户不会看到硬错误。

若要换供应商或自带 key，推荐用**环境变量覆盖**（自托管时优先用环境变量，不要依赖明文 key）：

```bash
# 例如改用 DeepSeek：
LLM_PROVIDER=deepseek  DEEPSEEK_API_KEY=你的Key  java -jar inner-cosmos-0.1.0.jar
# 或覆盖 GLM key：
GLM_API_KEY=你的Key  java -jar inner-cosmos-0.1.0.jar
# 也可强制离线 Mock：
LLM_PROVIDER=mock  java -jar inner-cosmos-0.1.0.jar
```

任一供应商无 key 或远程调用失败时，系统会沿失败转移链切换，最终回退到 Mock 风格回应，保证演示不中断。
