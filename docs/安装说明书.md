# Inner Cosmos 安装说明书

## 环境要求

- JDK 17 或更高版本。
- Maven 3.8 或更高版本。若本机没有 Maven，可使用 `scripts/run-dev.ps1` 自动下载本地 Maven。
- 默认演示不要求 MySQL，系统使用 H2 内存数据库。
- 如需 MySQL，请安装 MySQL 8 并创建 `inner_cosmos` 数据库。

## 方式一：运行已打包 JAR（推荐——老师无需安装 Maven）

```bash
java -jar inner-cosmos-0.1.0.jar
```

或在 Windows 上双击 `scripts/run-jar.bat`（如果提供了）。

启动后访问：

```text
http://localhost:8080/pages/index.html
```

默认账号：

```text
admin / admin123
demo / demo123
```

> **注意：** 默认使用 Mock LLM，无需任何 API Key 即可完整演示核心功能（对话、记忆沉淀、情感重力、共鸣体、慢信件等）。如需接入真实大模型，见下方"远程模型配置"。

## 方式二：从源码运行

## 默认运行

```powershell
.\scripts\run-dev.ps1
```

访问：

```text
http://localhost:8080/pages/index.html
```

默认账号：

```text
admin / admin123
demo / demo123
```

## MySQL 运行

创建数据库：

```sql
CREATE DATABASE inner_cosmos DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

修改 `src/main/resources/application-mysql.yml` 中的账号密码，然后运行：

```powershell
.\scripts\run-dev.ps1 -Profile mysql
```

## 远程模型配置

默认：

```yaml
llm:
  mode: mock
```

接入 DeepSeek / GLM / MiniMax 时切换为：

```yaml
llm:
  mode: remote
  provider: deepseek
  api-key: "你的 API Key"
  model: "deepseek-chat"
```

无 API Key 或远程失败时，系统会回退到 Mock 风格回应。
