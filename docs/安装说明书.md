# Inner Cosmos 安装说明书

## 环境要求

- **JDK 17 或更高版本**（项目以 Java 17 为编译目标，已在 JDK 17 与 JDK 21 上验证可运行）。
- 仅"从源码运行"时需要 Maven 3.8+；若本机没有 Maven，可使用 `scripts/run-dev.ps1` 自动下载本地 Maven，或直接用项目自带的 `./mvnw`。
- **默认演示无需 MySQL，也无需任何大模型 API Key**——系统使用 H2 文件数据库（落盘到 `./data/`）+ Mock LLM，开箱即用。
- 如需 MySQL，请安装 MySQL 8 并创建 `inner_cosmos` 数据库（见下方"MySQL 运行"）。

## 方式一：运行已打包 JAR（推荐——老师无需安装 Maven）

提交包内已提供可执行 JAR `inner-cosmos-0.1.0.jar` 与启动脚本 `run-jar.bat`（位于同一目录）。

- **Windows：** 双击 `run-jar.bat` 即可（脚本会自动定位同目录下的 JAR 并启动）。
- **任意系统命令行：** 在 JAR 所在目录执行：

```bash
java -jar inner-cosmos-0.1.0.jar
```

首次启动约需 10–20 秒（初始化 H2 数据库、建表与种子数据）。启动后访问：

```text
http://localhost:8080/pages/index.html
```

默认账号：

```text
admin / admin123
demo / demo123
```

> **注意：** 默认使用 Mock LLM，无需任何 API Key 即可完整演示核心功能（对话、记忆沉淀、情感重力、共鸣体、慢信件等）。如需接入真实大模型，见下方"远程模型配置"。

## 方式二：从源码运行

## 默认运行

```powershell
.\scripts\run-dev.ps1
```

访问：

```text
http://localhost:8080/pages/index.html
```

默认账号：

```text
admin / admin123
demo / demo123
```

## MySQL 运行

创建数据库：

```sql
CREATE DATABASE inner_cosmos DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

修改 `src/main/resources/application-mysql.yml` 中的账号密码，然后运行：

```powershell
.\scripts\run-dev.ps1 -Profile mysql
```

## 远程模型配置

默认：

```yaml
llm:
  mode: mock
```

接入 DeepSeek / GLM / MiniMax 时切换为：

```yaml
llm:
  mode: remote
  provider: deepseek
  api-key: "你的 API Key"
  model: "deepseek-chat"
```

无 API Key 或远程失败时，系统会回退到 Mock 风格回应。
