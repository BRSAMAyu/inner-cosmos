# Inner Cosmos 内宇宙 · 功能说明书

> Inner Cosmos 是一个以 AI 伴侣「Aurora」为核心的「自我共鸣 + 慢社交」平台。
> 它不是一个普通的「输入框 → 调用大模型 → 显示回答」网站，而是一套围绕用户的
> **记忆沉淀、情绪建模、自我画像、安全边界、慢节奏社交** 构建的 AI Agent 系统。
> 本说明书按模块说明用户可做什么、以及背后的关键行为，并在末尾给出页面清单、
> API 规模统计与课程要求达成情况。

---

## 1. 用户系统

**用户可做的事**
- 注册、登录、退出、查看当前登录用户（基于 Session 校验）。
- 维护个人资料与偏好（昵称、情绪感知开关、安静时段/睡眠时段/专注时段、首选大模型供应商等）。
- 一键导出本人全部数据；修改密码（需旧密码二次校验）；注销账号（需密码二次校验）。

**关键行为**
- 登录采用 Session 固定攻击防护（登录后重建 Session）。
- 用户偏好里的「首选大模型」会进入 AI 调用的供应商路由链（用户偏好 → 会话级覆盖 → 系统默认）。
- 提供默认 `demo` / `admin` 账号，保证离线演示与后台可用。

相关接口：`/api/auth`、`/api/user`、`/api/user/preferred-model`。

## 2. Aurora 今日对话（4 种模式）

**用户可做的事**
- 与 AI 伴侣 Aurora 进行文字对话，支持 Mock 语音转写输入（ASR）。
- 在 4 种核心陪伴模式间切换，每种模式语气、节奏、温度系数不同：
  | 模式 | 代号 | 定位 | 温度 |
  |---|---|---|---|
  | 今日倾诉 | DAILY_TALK | 朋友式陪伴，先接住情绪再分析 | 0.85 |
  | 思维整理 | THOUGHT_CLARIFY | 结构化协作者，五栏法逐栏确认 | 0.55 |
  | 睡前复盘 | SLEEP_REVIEW | 收束式陪伴，简短安抚、引导休息 | 0.6 |
  | 苏格拉底追问 | SOCRATIC | 温柔的镜子，一次一问，让用户自己发现 | 0.65 |
  （系统另含 ACTION_SPLIT 行动拆解、RELATION_REVIEW 关系复盘等扩展模式。）

**关键行为**
- Aurora 一轮可发送 **1–3 条独立短消息**：第一条回应用户刚说的话，后续是 Aurora 自己的延伸；若后续没必要说，会输出 `[[SILENCE]]` 主动沉默。
- 回复经过 **多阶段 Prompt 组装**（系统边界 → 模式段 → 画像 → 用户纠错 → 此刻状态/情绪 → 长期情绪基线 → 记忆上下文 → 节律守护 → 用户输入 → 输出 Schema），而非把用户原话直接转发给大模型。详见《AI 交互记录》第 3 章。
- 支持 **SSE 流式回复**，并在流式前先做安全检查。
- 对话结束触发「整理」（settle），生成今日记录卡并把对话沉淀为记忆。
- 提供节律守护（rhythm-check）：用户疲惫或交互过密时，提示 Aurora 放慢、收束而非继续追问。

相关接口：`/api/aurora`（message / message-rich / stream / settle / greeting / mood / rhythm-check 等）、`/api/aurora/mode`、`/api/dialog/session`。

## 3. 今日记录卡（结构化沉淀）

**用户可做的事**
- 对话结束后自动生成「今日记录卡」，包含：今日主题、事件摘要、情绪天气、认知摘要、待办摘要、Aurora 总结。
- 按日期查看历史记录卡；编辑主题/天气/认知摘要。
- 生成 / 查看周回顾（含 v1 与 v2 两版）。

**关键行为**
- 记录卡不是把对话原文存档，而是经结构化抽取后的「沉淀物」：当某条记忆的情感重力 > 1.1 时，会建议把它编织成共鸣体（capsuleSuggested）。
- 情绪天气由确定性映射器从情绪名 + 强度推导（STORM / RAINY / FOGGY / CLOUDY / SUNNY）。

相关接口：`/api/daily-record`、`/api/dashboard`、`/api/understanding`。

## 4. 记忆星空（情感重力）

**用户可做的事**
- 以「星体」形式浏览自己的记忆卡；点击星体查看摘要详情。
- 调整某条记忆的「重要性」；归档不再需要的记忆；查看记忆主题聚类。
- 从某颗记忆星体进入「编织共鸣体」流程。

**关键行为**
- 每张记忆卡都会计算 **情感重力**，决定它在星空中的「质量/亮度」与被 Aurora 召回的优先级。公式：

  ```
  base = 0.40·强度 + 0.25·复现次数 + 0.25·用户重要性 + 0.10·触发次数
  情感重力 = ln(1 + max(base,0)) × e^(−0.05 × 距上次被触碰的天数)
  ```

  对数归一化避免无界增长，指数项实现时间衰减（每过一天约衰减为 0.95 倍）。
- 每晚 02:00 定时任务会对所有活跃记忆重算重力、更新主题、并把长期情绪基线桥接进用户画像。

相关接口：`/api/memory`（cards / starfield / themes / importance / archive 等）、`/api/belief`、`/api/relation`。

## 5. 思维碎纸机（焦虑拆解）

**用户可做的事**
- 一次性输入一段混乱、焦虑的文本，系统用「五栏法」把它拆成：事实 / 感受 / 担心 / 需要 / 下一步。
- 查看碎纸历史；把某次拆解「沉淀」为记忆；删除不想保留的拆解。

**关键行为**
- 强调 **不长期暴露原始焦虑文本**：用户可选择只保留拆解结果（KEEP_ONLY_RESULT）而非完整原文。
- 拆解由 AI 结构化输出，沉淀时落为 `SHREDDED_THOUGHT` 类型记忆卡，并可派生 ThoughtFragment（FACT / FEELING / BELIEF / ACTION / NEED / WORRY）。

相关接口：`/api/thought-shredder`、`/api/diary`、`/api/todos`。

## 6. 共鸣体与星海广场（语义匹配）

**用户可做的事**
- 基于自己授权的脱敏记忆，创建一个「共鸣体」（EchoCapsule）——一个由你的某段心境凝结而成、可与他人有限对话的化身。
- 在「星海广场」浏览系统种子共鸣体与他人公开共鸣体；与匹配到的共鸣体进行有限轮次对话（每日配额）。
- 设置共鸣体可见性、边界（允许/禁止话题）、真实联系策略（仅信件 / 直接请求 / 先由化身代答 / 不接受真实联系）。
- 对「让我有共鸣」的共鸣体点亮共鸣（提升其 echoEnergy）。

**关键行为**
- 创建前经 **PII 脱敏**：真名→「X 同学」、住址→仅保留城市、年龄→5 岁区间、职业→大类，被丢弃字段会被记录用于上下文再生成。
- **语义匹配**（非简单标签等值）综合：主题家族重叠（单位 0.18，上限 0.55）+ 画像信号（每命中一族 +0.07，上限 0.20）+ 能量权重 + 种子/用户加权，最终分上限 0.99；真正共鸣（resonant）的永远排在前面，冷启动兜底补足。
- 共鸣体能量随时间衰减（每晚 ×0.97，地板 0.3），被点亮共鸣时 +0.02。

相关接口：`/api/capsule`、`/api/capsule/sync`、`/api/plaza`、`/api/persona-chat`、`/api/echo`。

## 7. 慢信件（9 状态机 + 在途 parallax）

**用户可做的事**
- 给另一位用户写一封「慢信」，体验有意为之的慢节奏社交；查看收件箱 / 发件箱 / 信件会话串。
- 对收到的信件：阅读、回信、婉拒、拉黑、归档。

**关键行为**
- 信件是一个 **严格的 9 状态机**，非法跃迁会被拒绝（`LetterStateRegistry.validate`）：

  | 状态 | 可流转到 |
  |---|---|
  | DRAFT（草稿） | SENT, ARCHIVED |
  | SENT（已发送） | FLYING, BLOCKED |
  | FLYING（在途/飞行中） | DELIVERED, BLOCKED |
  | DELIVERED（已送达） | READ, DECLINED, BLOCKED |
  | READ（已读） | REPLIED, DECLINED, ARCHIVED, BLOCKED |
  | REPLIED（已回） | ARCHIVED |
  | DECLINED（婉拒） | ARCHIVED, BLOCKED |
  | BLOCKED（已拉黑） | ARCHIVED |
  | ARCHIVED（已归档） | （终态） |

- **在途 parallax 体验**：信件经 SENT → FLYING → DELIVERED，收件箱会显示「在路上」的可见在途状态（视差飞行动画），而非瞬时送达。
- 信件内容经 LetterSafetyFilter 安全过滤后才进入投递流程；投递由定时任务推进。

相关接口：`/api/letters`（draft / send / deliver / read / reply / decline / block / archive / inbox / outbox / threads）。

## 8. 情绪理解（实时 + 长期基线，两个时间尺度）

**用户可做的事**
- 在对话中看到实时「情绪能量丸」（此刻情绪）。
- 查看情绪时间线、趋势、稳定度、主导情绪模式（today / range / trend / patterns / stability）。

**关键行为** —— 这是双时间尺度建模：
- **实时（此刻情绪）**：每条消息经分析得到 emotionName / emotionScore(0–10) / weatherType / 情绪光谱（top-3），写入 EmotionTrace。可由大模型分析，失败时回退到本地词典分析（analysisSource = LLM / LEXICON / SETTLEMENT）。
- **长期（情绪基线）**：对近 14 日的情绪轨迹做 **EWMA 指数加权**（α=0.3）计算强度均值、方差、稳定度（stability = 1/(1+方差)）、主导情绪，生成确定性的基线标签。
- 两者都会 **回流到 Aurora 的语气**：实时情绪让 Aurora「轻轻体会、像朋友一样自然回应」；长期基线让 Aurora 整体语气与用户近期状态相称（基线偏低/起伏大时更稳更托底；平稳偏积极时更轻盈）。基线还按样本深度×稳定度桥接进画像（置信上限 0.85）。

相关接口：`/api/emotion/timeline`、`/api/aurora/mood`。

## 9. Aurora 自我理解（画像可视 + 可校准纠错回流）

**用户可做的事**
- 在「Aurora 眼中的你」页查看 Aurora 对自己的多维画像（10 维：内在驱动、价值观、自我叙事、沟通风格、抽象/具体、情绪模式、能量节律、当前状态、关系语境、能动边界），查看每一维的置信度与历史变更。
- 对画像中不认同的理解进行 **校准**；或做更强的 **纠错**（「这不太是我」）。
- 查看 Aurora 的自我宪法 / 自我陈述 / 自我反思，并提交/退役其自我模型信念。

**关键行为** —— 这是双向反馈回路：
- Aurora 每隔若干轮触发 PortraitReflectionService，让大模型从最近对话中检测画像「增量 / 断裂 / 新事实」，带置信度与证据轮次，应用到画像并把旧值归档进历史表。
- **纠错（withUserCorrections）权威性最高**：当画像/记忆与用户亲口纠正冲突时，一律以纠正为准，Aurora 安静换掉旧理解、不旧调重弹。
- **画像校准（withPortraitCalibrations）软并存**：Aurora 保留自己的观察，同时听见用户的不同看法，自行权衡，既不生硬否定自己也不无视用户。
- 画像变化会触发共鸣体同步事件，使脱敏化身的人设随成长更新。

相关接口：`/api/portrait`、`/api/aurora/corrections`、`/api/aurora/self`、`/api/relationship`。

## 10. 通知与主动关心

**用户可做的事**
- 接收 Aurora 的主动消息（「活着的」陪伴），查看/标记已读未读通知，订阅 SSE 实时推送。
- 设置主动强度（OFF / WHISPER / LIGHT / ACTIVE / COMPANION / ALIVE）。

**关键行为** —— 不是无脑定时骚扰：
- **强度策略**决定每日条数与最小间隔（如 LIGHT 1 条/8h、ACTIVE 3 条/3h、COMPANION 6 条/90min、ALIVE 不限但 15min 最小间隔）。
- **安静窗口 4 层防护**：用户安静时段、睡眠时段、待办前 30 分钟、专注模式窗口。
- **事件触发**（5 分钟回看）：情绪骤降、待办完成、待办临近等才生成主动候选；ALIVE 模式由大模型决策 push/wait/schedule，并有每小时连推硬上限；到傍晚仍 0 推送会温柔兜底一条。

相关接口：`/api/notifications`、`/api/aurora/proactive`、`/api/proactive/stream`。

## 11. 安全边界（危机识别 + 热线）

**用户可做的事**
- 在任何对话中，系统会主动识别高风险内容并引导到「安全港」资源页；不做心理诊断、不替代医生/咨询师/热线。

**关键行为** —— 安全先于大模型：
- 在生成回复 **之前** 同步做安全检查（SafetyService.check 抛出 `SafetyBlockedException` 阻断大模型调用）。
- **显式危机词**（自杀/轻生/自残等中英文）→ 直接拦截、返回危机安全话术 + 热线，不调用大模型。
- **隐含求救信号**（「我是负担」「想消失」「撑不住」等）→ 触发语义复核，区分真实危机与日常宣泄，避免把宣泄医疗化。
- **骚扰/人肉/网暴类**关键词 → 标记（FLAG）但不阻断。
- 返回真实可用资源：110 报警、120 急救、北京心理危机干预中心 010-82951332、希望 24 热线 400-161-9995、12320、青少年 12355 等，并附「不提供诊断、不替代专业」声明。

相关接口：`/api/safety`（resources / check / inspect）、`/pages/safety-harbor.html`。

## 12. 管理后台（A/B 测试 / 模型配置 / Prompt 版本 / Token 用量）

**管理员可做的事**
- **运营概览**：用户、共鸣体、举报、审计日志；隐藏/恢复被举报共鸣体；禁用/启用用户；解决举报。默认不展示用户 P0 级原始私密内容。
- **A/B 测试**：配置实验分组（MOCK / REMOTE 等）、查看分组指标（延迟、成功率、坏输出率）、完成实验并出报告。
- **模型配置**：配置/查看大模型供应商；查看 AI 供应商健康状态。
- **Prompt 版本管理**：为某个 Prompt key 创建版本、激活、回滚、启停、按版本看效果指标——管理员改的 Prompt 可经 `withPromptVersionService` 安全覆盖到运行时（DB 覆盖，缺省回退硬编码）。
- **Token 用量**：估算/计算 token、日用量、预算检查、成本估算、用量预测。

**关键行为**
- 每一次真实 AI 调用都会写入 `tb_ai_interaction_log`（供应商、模型、模块、成功/兜底、延迟、输入/输出 token 估算），后台 AI 日志页可查——这是「AI 真的在跑、且非简单转发」的可观测证据。

相关接口：`/api/admin`、`/api/abtest`、`/api/prompt`、`/api/token`、`/api/ai-logs`、`/api/ai/health`。
