```yaml
status: READY

```

## 目标

确保生产构建不携带真实凭据、弱口令演示账号、Mock LLM 降级或不安全数据库缺省值。

## 已验证风险

- `application.yml` 存在不应入库的 Provider 凭据值；不得在日志、Issue 或证据中复制这些值；
- `application-prod.yml` 允许生产 fallback，数据库连接缺省不满足生产 TLS/Secret 要求；
- `MockDataInitializer` 是无条件组件，配置中的 seed 开关未形成可信生产阻断。

## Owned paths

- `src/main/resources/application*.yml`
- `src/main/java/com/innercosmos/config/MockDataInitializer.java`
- 与配置绑定、生产启动守卫相关的新类和测试
- `.gitignore`、secret scan 配置、CI 中对应扫描步骤

## 实施任务

1. 删除仓库中的凭据值，只保留环境变量/Secrets Manager 引用；
2. 用历史扫描确认影响范围，但输出必须脱敏；
3. 人类在 Provider 控制台吊销并轮换所有已暴露凭据；
4. 生产启动强制：fallback=false、seed=false、无默认弱密码、数据库 TLS；
5. 生产条件不满足时 fail closed，不静默切 Mock；
6. 增加配置单元测试和 `prod` 启动负向测试；
7. 构建镜像并扫描 Secret，检查日志不输出 Key。

## 验收

- S-01、S-04、S-05 通过；
- 仓库当前树、Git 历史报告、镜像与日志无可用 Secret；
- `prod` 缺关键配置时启动失败并给出不泄密错误；
- `prod` 不创建 admin/demo/river/cloud 等演示账号；
- 人类轮换记录只保存 Provider、时间和执行者，不保存新 Key。

## 回滚

配置改动可回滚到上一安全版本，但任何已轮换 Key 不得恢复；若生产启动守卫造成阻断，只能修复部署配置，不能重新启用不安全 fallback。
