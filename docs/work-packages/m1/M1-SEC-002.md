## 目标

阻止已登录用户用他人的 `sessionId` 触发、读取或改变 Goodbye 流程。

## 已验证风险

`GoodbyeController.trigger` 接收请求中的 `sessionId`；`GoodbyeOrchestrator.start` 加载会话后未验证 `session.userId == currentUserId` 即更新并继续编排。仅在 Controller 取到登录用户 ID 并不构成对象级授权。

## Owned paths

- `src/main/java/com/innercosmos/controller/GoodbyeController.java`
- `src/main/java/com/innercosmos/ai/goodbye/GoodbyeOrchestrator.java`
- 对应 Goodbye ownership 测试文件

## 实施任务

1. 在 service/orchestrator 事务边界做 owner 校验，controller 可保留防御性校验但不能成为唯一防线；
2. 未找到与非 owner 使用一致的对外错误语义，避免对象枚举；
3. 校验失败不得更新会话、创建事件、调用 LLM 或触发异步副作用；
4. 覆盖 owner、non-owner、admin（若无明确授权仍拒绝）、不存在 ID、并发状态变化；
5. 搜索 Goodbye 其他按 ID 操作并补齐同一 ownership policy。

## 验收

- S-02 对 Goodbye 部分通过；
- non-owner 测试证明数据库无变化、无下游调用；
- owner 正常流程及幂等测试不退化；
- 全量回归通过。

## 回滚

若兼容性问题出现，保留 service 层授权，回滚 controller 表现层变化；绝不回滚对象所有权校验。
