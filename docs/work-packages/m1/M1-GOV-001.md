## 目标

让所有 Agent 对 M1 的产品目标、范围、决策权威、工作包状态和证据格式只有一个解释。

## Owned paths

- `对齐文档/00-项目理解与云原生产品化总纲.md`
- `对齐文档/05-Agent全自主开发与交付周期设计.md`

- `docs/work-packages/m1/**`
- 新增 `docs/adr/**`、`evidence/m1/README.md` 模板文件

## 实施任务

1. 建立 ADR 模板，至少包含 context、decision、alternatives、consequences、rollback、evidence；
2. 建立 evidence manifest schema，要求 SHA、环境、命令、时间、原始产物校验值；
3. 建立机器可读的 M1 acceptance ledger，录入 P/S/D/R/K/O/V 条目；
4. 检查 `00`—`07` 对 M1 是否存在冲突，冲突必须显式标记 superseded；
5. 将 H2/H3 项移入 parked backlog，不得删除长期愿景。

## 不做

- 不改业务代码；
- 不新增架构愿望；
- 不替团队批准真实生产、合规或预算决策。

## 验收

- 每个 M1 acceptance ID 在 ledger 中唯一；
- 每个 READY 包都有依赖、owned paths、测试、证据和回滚；
- 文档链接检查通过；
- 独立 reviewer 能仅凭这些文件说明下一包、阻断和完成定义。

## 回滚

仅回滚本包新增的治理模板和未被后续包引用的链接；不得回滚已签署 ADR，需用 superseding ADR 取代。
