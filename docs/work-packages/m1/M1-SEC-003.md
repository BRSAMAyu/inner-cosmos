## 目标

阻止用户用他人的 candidate ID 将 Aurora Self reflection 提交到自己的模型或造成跨用户副作用。

## 已验证风险

`AuroraSelfContinuityServiceImpl.commitToModel(userId, candidateId, ...)` 按 candidate ID 加载 reflection 后未验证其 owner；同类 `retireModel`、`dismissCandidate` 已有 owner 校验，语义不一致。

## Owned paths

- `src/main/java/com/innercosmos/controller/AuroraSelfController.java`
- `src/main/java/com/innercosmos/service/impl/AuroraSelfContinuityServiceImpl.java`
- 对应 Aurora Self ownership 测试文件

## 实施任务

1. 在 service 事务边界统一 candidate/model 的 owner 校验；
2. non-owner 与不存在对象使用不可枚举的统一错误；
3. 校验失败不得创建 self model、更新 candidate、触发 capsule sync 或其他事件；
4. 增加 commit、dismiss、retire 的 owner/non-owner 矩阵测试；
5. 只修复 ownership 与跨用户副作用；不得删除、降级或隐藏 Aurora Self / Constitution / Emergence、人格、关系状态、演化或用户体验能力。相关产品方向等待 Innovation ADR。

## 验收

- S-02 对 Aurora Self 部分通过；
- non-owner 测试证明所有相关表无变化且无异步副作用；
- 现有 Aurora Self 测试与全量回归通过。


## 回滚

不得回滚 service 层 owner 校验；如 API 兼容受影响，只允许调整统一错误映射。
