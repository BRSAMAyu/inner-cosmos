# 导师答辩：画像编译机制与共鸣体 Genome 编译器

> 本文档用于系统性介绍 Inner Cosmos 的数字人设编译机制、Genome v3 中间表示 (IR) 设计以及基于声线/场景的可追溯性 (Provenance) 体系。

---

## 1. 为什么需要画像编译器？(The Genome Paradigm)

在传统的大模型应用中，如果我们想让一个 AI 代表某个用户，通常是简单粗暴地将用户所有的聊天记录或记忆卡片塞进 System Prompt 或者作为 RAG 检索喂给大模型。这种做法在学术和工程上存在严重的弊端：
1.  **数据冗余与上下文污染**：原始记忆包含了大量的流水账和无关细节，会导致 LLM 理解偏差。
2.  **严重的隐私泄露风险**：一旦黑客通过 Prompt 注入，用户毫无脱敏的原始细节会全盘泄露。
3.  **零可追溯性**：AI 会因为深度学习的“黑盒”本质，瞎编出不属于该用户的人格特征，而开发者和用户无法审计为什么 AI 会这样说话。

为了解决这个问题，Inner Cosmos 独创了 **Genome (基因) 编译体系**：
*   它不直接传输用户的记忆卡片本身。
*   它将经过授权的结构化记忆作为原料，通过画像编译器（Genome Compiler），在本地对其进行“降维”和“特征提取”，生成高度抽象、脱敏的数字人格模型 —— **`CapsuleGenomeVersion`**。

## 2. Genome v3 的中间表示 (IR - Intermediate Representation) 设计

编译出来的 Genome 包含三部分结构化的中间表示数据，它们以加密/格式化的 JSON 字符串形式持久化在数据库的 `tb_capsule_genome_version` 中：

1.  **`authorizationSnapshotJson` (授权与同意快照)**：
    精确地记录了当前版本共鸣体编译时所使用的所有 `memoryId` 列表、原始记忆的版本号以及对应的同意授权证书（`DataUseGrant` 的 ID 和版本号），确保数据合规可审计。
2.  **`styleProfileJson` (声线与人格特质) - 脱敏表达层**：
    包含推导出的语言风格、情绪稳定系数、核心偏好、以及人设声线（Voice Descriptors，例如：“温和、理性、喜欢以倾听为主”）。
3.  **`contextPreviewJson` (场景与话题边界) - 语义过滤层**：
    定义了该共鸣体具备哪些回忆场景（如：对学术研究的探讨、日常运动经历），以及明确的话题准入名单与黑名单规则。

## 3. 基于 `voiceEvidence` 的可追溯性 (Provenance) 体系

为了保证数字人格的真实度，彻底解决 AI 编造人设（幻觉）的硬伤，我们的 Genome 编译器引入了**物理层面的证据链审计**：

*   **场景追溯 (Scene Provenance)**：
    在 `contextPreviewJson` 中的每一个推导出的交互场景，都必须附带一个 `memories` 来源数组，里面包含了精确的 `{memoryId, sourceVersion, confidence}` 映射。
*   **声线特质追溯 (Voice Evidence)**：
    在 `styleProfileJson` 的每个人格声线描述中，都必须携带一个 `voiceEvidence` 映射，它定义了 `theme -> [memoryIds]`，明确指出：**为什么共鸣体会体现出“热爱阅读”或者“性格慢热”这一特质，是由哪几条具体的原始记忆推理得出的**。
*   **学术意义**：
    这极大地提升了系统的 Groundedness (可信接地度)。当共鸣体被拿去和别人对话时，系统可以根据 `voiceEvidence`，动态召回最相关的记忆片段，杜绝了 AI 脱离用户真实过往而凭空说假话的行为。

## 4. 可信安全门禁与回归验证 (Groundedness Testing)

系统设计了严密的回归测试机制 —— [GenomeCompilerGroundednessEvaluationTest.java](file:///d:/code/inner%20cosmos/src/test/java/com/innercosmos/evaluation/GenomeCompilerGroundednessEvaluationTest.java) 来自动化保证这套安全底线：
*   **泄漏测试断言**：
    在测试中，我们向编译器传入正常授权的记忆卡片，并故意掺杂入一条越权的卡片（例如：属于其他用户的记忆卡片，或者被标记为“禁止外部处理/仿真测试专用”的卡片）。
*   **熔断拦截**：
    测试会自动断言：最终生成的 `voiceEvidence` 与 `scenes[].memories` 中，**绝对不能**包含该条越权卡片的 citation（引用）。一旦出现越权引用，测试会立刻抛出 `ungroundedCitations` 错误并熔断部署流程，保障了云原生发布阶段的隐私安全。
