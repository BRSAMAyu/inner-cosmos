# 导师答辩：数据管线与隐私生命周期

> 本文档用于系统性介绍 Inner Cosmos 的 P0-P3 数据隐私分层机制、记忆沉淀数据管线、情感重力计算以及级联遗忘（Cascade Forgetting）的完整生命周期。

---

## 1. 核心设计：P0 至 P3 数据隐私分层机制

为保障用户的核心隐私，系统将数据按照敏感程度划分为 4 个层级，在数据库底层和代码访问逻辑上实行物理上的访问隔离：

```text
[ P0 原始对话 ]  --->  [ P1 结构化记忆 ]  --->  [ P2 共鸣体基因 ]  --->  [ P3 社交数据 ]
(私密, 不出机)          (脱敏, 纠错暂存)          (高度抽象, 授权)        (慢信, 社交网络)
```

*   **P0（原始对话层）**：`tb_dialog_session`、`tb_dialog_message`。记录用户与 Aurora 伴侣最私密的原始对话。**绝对不能**被任何共鸣体、外部游客或管理员所读取。
*   **P1（结构化记忆层）**：`tb_memory_card`（事实、信念、行动等）、`tb_thought_fragment`、`tb_emotion_trace`（情绪痕迹）。由原始对话脱敏提炼而成，并需经过用户在前端的 Staging 审核纠错后，才标记为 `ACTIVE` 状态。
*   **P1.5（数据同意授权）**：`tb_data_use_grant`。用于精准判定哪一部分 P1 级记忆卡片可以被哪一个 P2 级共鸣体使用。
*   **P2（共鸣体层）**：`tb_echo_capsule`、`tb_capsule_genome_version`。不包含原始对话与直接记忆卡片，只包含高度脱敏提炼的“声线特征”和“话题边界说明”。对外提供只读的 AI Persona 对话服务。
*   **P3（公开社交层）**：`tb_slow_letter`（慢信）、`tb_persona_chat_message`（访客与共鸣体对话记录）。用于支撑外部用户之间的有限对话与慢交友，不携带任何原作者的个人标识信息。

## 2. 记忆沉淀数据管线 (Memory Ingestion Pipeline)

当用户与 AI 伴侣 Aurora 的一轮对话结束时，系统会触发 `DialogFinishedEvent`，通过观察者模式（Observer Pattern）异步驱动以下数据管线：
1.  **事实与信念提取 (Memory Extraction)**：
    大模型或精度优先提取器扫描对话，提炼出数条 `ThoughtFragment`（包含事实描述、用户潜在的信念与行动意图）。
2.  **Staging 暂存与人机协同纠错 (User-Correction Path)**：
    新提取的数据不会直接生效。它们会作为 `STAGED` 状态的待审核事实，呈现在前端纠错面板。用户可以编辑文本、修正 AI 提取出的偏差，点击 `Confirm` 后才转化为 `ACTIVE` 的 `MemoryCard`，或者点击 `Dismiss` 一键丢弃。**确保了数字人格的主动权牢牢握在人类手里**。
3.  **情感痕迹记录 (Emotion Trace)**：
    分析本轮对话的整体情绪走向，计算情绪分值（喜悦、悲伤、焦虑分值），存入 `tb_emotion_trace`。

## 3. 情感重力算法与星空聚类 (Gravity & Clustering)

*   **情感重力 (`GravityService`)**：
    记忆是有权重的。系统引入了“情感重力”计算：
    $$\text{Gravity} = f(\text{EmotionIntensity}, \text{Recency}, \text{UserAccessCount})$$
    结合记忆的情感强烈度、时间衰减系数（符合艾宾浩斯遗忘曲线）以及用户主动查看/纠错该记忆的频率，计算出每张卡片的“重力”分值。
*   **记忆星空渲染**：
    前端读取这些记忆卡片的坐标和重力，将它们渲染为天体。重力越大的记忆，在星空里体积越大、亮度越高，代表其对用户的心理影响越深。同时，系统在夜间调度任务中，会根据主题相似度对卡片进行“星云聚类”，将零碎的记忆聚合为长期主题。

## 4. 级联物理遗忘机制 (Cascade Forgetting)

为了彻底履行 GDPR 规范中的“被遗忘权（Right to be Forgotten）”，项目在底层设计并实现了**级联遗忘 (Cascade Forgetting)** 算法，确保数据擦除的彻底性：
*   **单点删除的隐患**：如果在页面上点击删除某条记忆，但该记忆已经被拿去编译了共鸣体（已发布到广场），或者已经缓存进了对话 prompt 中，那么单点删除就会失效，共鸣体依然可能说出被删记忆的信息。
*   **我们如何实现级联物理擦除**：
    当用户发起遗忘（FORGET）某一 MemoryCard 时，在同一个数据库事务中：
    1.  物理删除该 `MemoryCard` 及关联的 `ThoughtFragment` 记录。
    2.  扫描所有引用了该记忆卡片的已编译共鸣体，获取其 `capsuleId`。
    3.  **彻底下架共鸣体**：立即将对应的共鸣体状态修改为 `NEEDS_REVIEW`，将其可见性设为私有（`is_public = false`），并从公共星海广场中撤出。
    4.  **抹除基因特质**：将已编译的 `styleProfileJson` 和 `contextPreviewJson` 中有关该记忆生成的声线、场景描述直接抹去，强制触发 Genome 的下一次重新编译，确保受影响的历史 Genome 永久失效。
    5.  **撤销使用权授权**：在 `tb_data_use_grant` 中将该记忆对共鸣体的所有授权记录标记为已撤回（`REVOKED`）。
    *结论*：通过事务的级联锁死，保障了隐私物理抹除的绝对一致性。
