# Inner Cosmos 完全体产品愿景与功能规格

> 文档性质：L1 产品目标规范
> 状态：AUTHORITATIVE
> 目标版本：Complete Product / Singapore Release Candidate
> 关联：`08-Aurora生命感与共鸣智能创新架构.md`、`10`—`12`、`docs/goal/complete-product-acceptance.yml`

## 0. 产品一句话

Inner Cosmos 是一个具有长期理解、时间感和关系连续性的 AI 自我共鸣与慢社交系统：Aurora 陪用户表达与行动，记忆星空让用户看见自己的变化，共鸣体在明确授权下把内在世界转化为更有质量的人际连接。

它不是通用聊天 UI、AI 日记、心理诊断工具或陌生人速配。其不可替代性来自一条完整的数据—智能—体验飞轮：

```text
自然表达与生活轨迹
  → Aurora 当下承接、澄清、陪伴与行动支持
  → 可纠正的记忆、画像、主题、关系与心理模型
  → 时间中的变化、矛盾和成长被可视化
  → 用户选择并授权形成高保真共鸣体
  → 多策略发现、共鸣对话、慢信与真实连接
  → 新互动在权限边界内反哺用户理解
```

## 1. 市场位置与产品原则

### 1.1 要占据的位置

Inner Cosmos 位于通用 AI 聊天、日记/情绪记录、心理自助内容与社交产品之间，但不等同于其中任何一类：

- 比通用聊天更连续：理解跨越数周和数月，能解释记忆来源、变化和不确定性。
- 比日记更有回应：Aurora 会追问、组合、记住、主动回来，也能帮助用户把洞察变成行动。
- 比普通社交更低压力：先与经过授权的共鸣体形成理解，再决定是否接近真人。
- 比单纯心理内容更个体化：心理学机制被做成有证据、有边界、可调用的 Skill，而非模板文章。
- 比传统 companion 更现实导向：拟人感服务于被理解和关系连续性，不以制造依赖或取代真人为商业目标。

### 1.2 六条产品宪法

1. **被理解，而不是被归类**：画像是可纠正、带置信度和来源的动态假设，不能把用户锁进标签。
2. **生命感，而不是随机扮演**：Aurora 的主动性、风格与关系演化来自可维护的内部状态和系统机制。
3. **创新不以阉割换安全**：用分层授权、最小披露、可撤销、审计和质量评测治理高能力。
4. **从内在走向现实**：最好的结果不仅是停留更久，也包括更清楚地行动、求助、休息和建立真实连接。
5. **高质量优先于低推理成本**：核心 demo 和关键体验可以使用双核、多 critic、多阶段编译和更昂贵模型。
6. **用户拥有自己的内在资产**：记忆、画像、共鸣体和导出/删除权属于用户，系统必须解释如何生成和使用。

## 2. 核心用户与 Job to be Done

### 2.1 核心用户

- 想被持续理解、但不愿从头反复解释自己的年轻成年人。
- 有表达和反思习惯，却难以从碎片中看见长期模式的人。
- 对即时社交疲惫，希望先理解、后连接的人。
- 处于跨文化、留学、迁移或人生变化期，需要低压力陪伴和自我整理的人。
- 对 AI companion 有兴趣，但要求更真实、更可控、更有深度的早期用户。

### 2.2 核心任务

| 情境 | 用户真正要完成的任务 | 产品成功信号 |
|---|---|---|
| 情绪或思绪混乱 | 先被接住，再找到可说清的核心 | 对话后更清楚而非只觉得“回复温柔” |
| 长期反复 | 看见主题、触发、关系和行为如何变化 | 用户能纠正并认可形成的时间线 |
| 需要陪伴 | 希望有人记得、会回来、知道现在是什么时候 | 主动消息恰当、可控且有上下文 |
| 想了解自己 | 获得多角度但不武断的心理与行为洞察 | 洞察可解释、可选择深入、可行动 |
| 想认识人 | 找到“能共鸣”或“能互补”的人而非流量匹配 | 从共鸣体到慢信/真人连接的质量 |
| 想保留边界 | 决定什么能被模型、共鸣体或他人使用 | 授权清晰、随时撤回、撤回确实生效 |

## 3. 完全体产品支柱

### P1. Aurora：一个具有连续性的相遇对象

Aurora 必须表现为同一个持续存在、能理解时间和关系上下文的主体，而不是每次请求重新拼出的客服 Prompt。

必需体验：

- 一次回应可被编排为若干条自然消息，允许短停顿、补充、犹豫、重述和随后想起。
- 用户新消息可打断其思考或输出；Aurora 能停止、吸收新信息并重规划，而非把旧答案吐完。
- 用户可以看见“正在听、在想、准备说、决定稍后回来”等真实系统状态，但不暴露伪造的模型思维链。
- 快核承接当下情绪与社交节奏，慢核做记忆、矛盾、关系、计划和安全反思；关键回应由编排器合成。
- Aurora 有可版本化的 Self Genome、Constitution、Relationship State 与 Emergence Proposal，能够演化、评测和回滚。
- Aurora 知道时区、一天时段、对话间隔、承诺和未来事件，可在满足用户契约时主动唤醒。
- 主动性不是固定每日一句：支持陪伴、提醒、追问、庆祝、未完成事项回访、沉默关照和环境触发等意图。
- 用户可调节主动程度、安静时段、主题、渠道、关系距离、是否允许多消息与何时仅在应用内出现。

体验底线：Aurora 不应永远赞同、机械复述、重复免责声明、凭空宣称记忆、伪造外部行动或用黏着话术阻止用户离开。

### P2. Inner Model：可见、可纠正的长期自我理解

系统从对话、记录、选择、反馈与授权数据中建立长期模型，但必须把“事实”“推断”“当前假设”分开。

完全体画像至少覆盖：

- 稳定事实与生活背景；兴趣、偏好、厌恶和审美。
- 重要人物、关系角色、互动模式与边界。
- 价值观、身份叙事、信念、愿望、目标与冲突。
- 情绪基线、触发、恢复方式、身体/睡眠等用户自愿提供的生活信号。
- 沟通风格、节奏、常用表达、幽默、习惯和可接受的陪伴方式。
- 当前议题、未解决问题、承诺、行动、阶段变化与长期 trajectory。
- 模型来源、置信度、时间衰减、争议状态和用户修订历史。

用户必须能够：查看“为什么系统这样理解我”、确认/修正/删除、区分私密层级、暂停学习、导出数据，并预览某项信息将如何影响 Aurora 或共鸣体。

### P3. Memory Cosmos：让时间和变化可被感知

记忆星空不是卡片墙的视觉包装，而是用户长期模型的可探索界面。

- 星体属性由情感重力、时间、复现、关系、主题和用户标记共同决定。
- 支持时间旅行、主题星座、人物轨道、情绪天气、未解决引力、转折点与成长轨迹。
- 每个洞察可追溯到授权来源，允许纠正后重新计算相关视图。
- 新记忆会被去重、合并、强化、衰减、冲突标记或遗忘；系统显示变化而非永远追加。
- “本周变化与未解决问题”替代模板周报：指出变化、证据、不确定性和可选行动。
- 星空兼具情绪美学和信息可读性，在低性能、低动效和无音频模式下仍然可用。

### P4. Psychology Skill Studio：可扩展的自我探索能力

心理学能力以版本化 Skill/Plugin 注册到 Aurora 和用户界面，既可由 Agent 在合适时机建议，也可由用户显式启动。

首批能力族：

- 情绪标注与调节偏好探索。
- 价值澄清、需求识别、目标冲突和决策反思。
- 依恋/沟通/应对方式等非诊断性模式探索。
- 经合法授权采用的标准化量表或自评，清楚显示来源、用途、限制和结果解释。
- 行为实验、微行动计划、复盘和变化追踪。
- 危机识别与本地资源导航，但不把危机用户留在生成式陪伴闭环中。

每个 Skill 必须声明输入范围、适用人群、证据来源、版本、输出 schema、禁忌、风险等级、何时转介、数据保留和评测集。任何临床诊断或治疗宣称均不在默认产品范围内。

### P5. Echo Capsule：高保真、可授权的动态人格资产

共鸣体不是静态卡片，也不是把用户所有资料塞进 Prompt。它由 Capsule Genome Compiler 从授权数据编译：

- 核心事实、价值、兴趣、关系视角、语言风格、习惯、癖好、典型反应、情绪节奏与边界。
- 分层记忆和按场景检索；只向当前问题注入最相关且允许的信息。
- Planner 决定回应意图，Speaker 生成候选，事实/风格/边界/泄露 critic 评分，reranker 选择最终回应。
- 支持版本、预览、沙盒试聊、差异解释、用户修正、撤回、暂停与销毁。
- 保持“这是由某位用户授权形成的 AI 共鸣体”这一透明身份，不冒充真人在线或虚构实时行为。

其高保真模拟技术可以成为独立能力资产，用于获得授权的产品测试、角色模拟或研究；任何二次用途必须使用隔离的数据合同，不能把个人共鸣体默认转作训练或商业模拟。

### P6. Resonance & Slow Social：先理解，再接近真人

发现不是单一余弦相似度排行，而是可解释的多目标策略：

- `MIRROR`：相似经历、价值或处境带来的被理解感。
- `COMPLEMENT`：差异兴趣、能力或视角带来的互补。
- `GROWTH_EDGE`：在边界内接近可能帮助成长的人。
- `SERENDIPITY`：保留多样性和偶遇，避免同质回音室。
- `CONTEXTUAL`：围绕当前主题、阶段或活动匹配。

匹配使用多向量空间、显式偏好、边界、质量与新鲜度，不把心理脆弱性当作广告/操纵特征。用户可查看“为何相遇”、调整策略或隐藏某类信号。

共鸣体对话额度应防滥用但不破坏体验：陌生发现阶段可采用宽松的防刷和公平额度；当双方形成连接或用户进入自己的资产，不应以任意轮次把关系锁死。慢信是从共鸣验证到真人关系的节奏机制，支持延迟、草稿、边界、拒绝、举报、静音和退出。

### P7. Trust, Mobile & Singapore：真实产品而非实验孤岛

- Web/PWA 与 Capacitor App 共享核心体验；推送、麦克风、安全存储、深链和离线草稿形成移动端增量价值。
- 中英双语、`Asia/Singapore` 时区、本地危机资源、文化适配和无障碍是发布候选必需能力。
- 用户拥有清楚的隐私中心：同意、数据层级、跨境 Provider、下载、删除、撤回、共鸣体授权和主动消息控制。
- 对新加坡发行的 PDPA、跨境传输、商店、HSA 边界、税务与运营要求，采用 `04` 的研究结论并在发布前重新核验、人工签字。

## 4. 端到端用户旅程

### J1. 第一次相遇

1. 以价值体验开场，而非先填冗长问卷。
2. 用户选择语言、称呼、时区、陪伴强度、通知和敏感数据同意。
3. Aurora 通过一段自然对话学习“希望如何被陪伴”，展示一次可撤回的记忆预览。
4. 用户看见：哪些内容仅用于当前对话、哪些进入长期记忆、哪些绝不会进入共鸣体。
5. 首次体验在十分钟内完成“被回应—形成一颗星—看见为什么”的闭环。

### J2. 日常对话与被主动想起

1. 用户用文字、语音或快捷情绪入口开始。
2. 界面表达 Aurora 的倾听、思考和多消息节奏；用户随时打断或让其继续。
3. 会话后系统建议记忆、画像更新、待办和可选 Skill，用户可一键修正。
4. Temporal Engine 依据承诺、间隔、时段和用户契约建立 WakeIntent。
5. 到时先经过相关性、边界、风险、预算和安静时段决策，再通过应用内或推送联系。

### J3. 看见自己的变化

1. 用户进入 Inner Cosmos 首页，看见当前情绪天气、主要星座和未解决引力。
2. 可沿时间、主题、人物或关系进入轨迹。
3. 点击洞察看到来源、系统解释、置信度和替代解释。
4. 修正一项理解后，相关画像、星空和共鸣体进入可预览的重算流程。

### J4. 显式自我探索

1. 用户从 Skill Studio 选择一个问题或接受 Aurora 的低压力建议。
2. 产品解释用途、耗时、数据范围和非诊断边界。
3. Skill 以对话、卡片或量表完成，允许暂停。
4. 输出包含发现、证据、不确定性、可选行动和何时寻求真人支持。
5. 用户决定哪些结果进入画像或仅保留本次。

### J5. 创造并验证共鸣体

1. 用户选择目标：被理解、结识同类、寻找互补、表达某一侧面或建立模拟器。
2. 授权向导按信息维度而非数据库表展示可选内容。
3. Compiler 生成 Genome 与覆盖报告，指出缺失、冲突和高风险项。
4. 用户在沙盒以多个典型问题试聊，比较“像/不像/泄露/过度推断”。
5. 发布后持续获得质量反馈；任何源数据撤回都会触发失效与重新编译。

### J6. 从共鸣到真人

1. 用户选择匹配策略并浏览有解释、有边界的共鸣体。
2. 与共鸣体自然交谈，系统在不打断情绪流的前提下标明身份与授权范围。
3. 双方达到连接条件后，可发送慢信；接收者有充分的预览、拒绝和屏蔽权。
4. 只有双方明确同意才开放更多真人信息或实时沟通。
5. 关系结束后可归档、撤回授权，并保留必要的安全审计而不继续暴露内容。

## 5. 信息架构

面向用户的顶层空间收敛为五个，不再让数十个物理页面成为认知模型：

1. **Today / Aurora**：对话、当前状态、主动消息和下一步。
2. **Inner Cosmos**：星空、时间线、画像、洞察、Skill 和数据纠正。
3. **Resonance**：共鸣体创建、探索、匹配、会话与连接。
4. **Inbox**：慢信、通知、请求和系统解释。
5. **Me**：账户、关系契约、隐私、设备、导出删除、语言与外观。

运营后台、研究评测和开发控制台是独立权限域，不混入普通用户导航。

## 6. 功能完整性与优先关系

| 能力 | 完全体要求 | 不可接受的伪完成 |
|---|---|---|
| Aurora 对话 | 编排事件、双核、打断、停止、多消息、时间感、长期上下文 | 单次请求返回一段文本 |
| 主动性 | WakeIntent、用户契约、相关性与风险决策、可观察反馈 | 固定 cron 群发问候 |
| Self/Emergence | 结构化状态、提案、评测、版本与回滚 | Prompt 中写“你会成长” |
| 记忆 | 获取准、更新、合并、冲突、淘汰、复用、可纠正 | 向量库只增不减 |
| 用户画像 | 多维、有来源/置信度/时效、可解释与纠正 | 一份不可追溯的 LLM 总结 |
| 共鸣体 | 编译、检索、规划、生成、critic、沙盒和撤回 | 静态 persona 卡 + 超长 Prompt |
| 匹配 | 多空间、多策略、边界与解释、离线/在线评测 | 单一 embedding 最近邻 |
| 心理 Skill | 有来源、schema、边界、风险和评测的插件运行时 | 随机让模型做心理诊断 |
| 星空 | 数据语义与视觉联动、时间/主题/人物探索 | 装饰性粒子背景 |
| 移动端 | PWA + Capacitor 增量能力与发布证据 | WebView 截图即称 App 完成 |
| 云原生 | 状态外置、幂等、异步、弹性、恢复、观测 | 只有 Deployment YAML |

## 7. 安全、伦理与体验的共同边界

- 明确 AI 身份和能力边界，但不在每轮用模板免责声明破坏交流。
- 危机响应按风险分层：承接、鼓励现实支持、显示本地资源；必要时中止普通生成流程。默认不声称人工监控或紧急救援。
- 拟人化允许存在，产品不能虚构真人意识、现实行动或以威胁/内疚阻止退出。
- 主动消息以用户契约为基础；不用“连续签到”“她会孤单”等机制制造负罪感。
- 共鸣体高保真与隐私同时成立：数据最小化、用途绑定、可撤销和输出泄露评测是技术要求。
- 心理模型不得用于歧视性排序、价格差异、广告操纵或未经同意的脆弱性利用。
- 删除与撤回必须传播至派生画像、向量、缓存、共鸣体和后续任务；必要审计采用最小保留。

## 8. 产品质量指标

北极星不是消息数或停留时长，而是“用户是否在安全边界内获得持续自我理解，并形成有质量的行动或连接”。指标分四组：

- **理解质量**：记忆确认率、画像纠正率、来源可解释率、长期一致性、洞察有用度。
- **关系质量**：Aurora 连续性感知、主动消息恰当率、打断恢复成功率、共鸣体相似/边界评分、慢信积极回应率。
- **现实价值**：自报清晰度变化、行动完成、愿意寻求真人支持、连接后质量，而非纯黏性。
- **信任质量**：撤回成功率、隐私理解度、危机路径成功率、投诉/泄露/越权率、数据删除 SLA。

所有指标必须按语言、设备、新老用户、主动性偏好和高风险场景分层，避免总体均值掩盖伤害。

## 9. 课程展示与商业产品的双重成果

展示版必须在 8—12 分钟内呈现真正差异：

1. Aurora 自然多消息回应，被用户打断后停止并改变计划。
2. 她记得一段长期上下文，在约定时间主动回来，用户能看见并调节原因。
3. 一段混乱表达变成可纠正的记忆与画像更新，并在星空中形成变化。
4. 用户启动一个心理 Skill，得到有来源、有边界的个体化洞察。
5. 系统编译高保真共鸣体，沙盒比较其风格与事实一致性。
6. 通过相似/互补策略发现另一共鸣体，完成对话和慢信。
7. 同一产品运行在移动端和 EKS，可展示 trace、弹性或故障恢复证据。

商业候选则必须在此之上满足 `12` 的隐私、运营、恢复、商店和人工门禁；不得用漂亮 demo 代替真实责任能力。

## 10. 非目标与延后条件

- 不为课程炫技预先拆成大量微服务。
- 不默认引入 Kafka、Service Mesh、图数据库、多区域 active-active 或自研模型训练。
- 不做临床诊断、治疗替代或未经批准的医疗器械宣称。
- 不做公开流量社交、无限陌生人私信或基于脆弱性的推荐。
- 不要求一开始拥有所有原生 UI，但最终移动发布候选必须有 Capacitor 原生能力和商店证据。
- 技术只有在现有选择无法满足量化目标时升级；创新重点投入 AI runtime、记忆画像、共鸣体、匹配和体验，而非基础设施堆砌。
