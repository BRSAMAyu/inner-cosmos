# 内心宇宙 → 新加坡发行开发研究

> 本文由 4 个 agent 并行 web 研究(2026-07 核查)后汇总:法规数据合规 / 服务器网站 / 应用商店医疗边界 / 商业税务。只提供信息、现状、选项与权衡,不给执行步骤,不构成新加坡执业法律意见。证据附官方 URL。
> 项目背景:中文 AI 心理情绪陪伴 Web/App(内心宇宙),后端调**中国大陆 LLM**(智谱 GLM / MiniMax / DeepSeek / 小米 MiMo),计划在新加坡发行(服务器/网站/App 面向新加坡)。

## 0. 核心结论(先读)

**新加坡是本项目的良好发行地,且现阶段几乎零成本即可启动:**

1. 🟢 **无需任何本地备案**(决定性优势 vs 中国):新加坡走**全球 Apple App Store 新加坡区 + Google Play 新加坡**,不存在中国式 ICP/APP 备案、MIIT 注册、软著要求,也**不需要新加坡本地法人**。一个标准开发者账号即可上架。

2. 🟢 **非医疗陪伴 App 不是医疗器械,无需 HSA 注册**:判定钥匙是"预期用途 + 医疗声称"。本项目"情绪陪伴 + 危机引导热线"落在非医疗器械一侧。**红线措辞 = 诊断/治疗/疗效/CDSS** —— 一行错误营销文案就能把产品推成"未经注册的医疗器械"。

3. 🟢 **现阶段以中国个人/实体直接发行完全可行,GST 由 Apple/Google 代收**:不用注册新加坡公司、不用碰 GST、不用申请补贴。**零合规负担启动**。

4. 🟡 **唯一核心合规问题 = PDPA 跨境传输到中国 LLM**:把新加坡用户的**心理/情绪/记忆数据**(实务最高敏感)发给中国 LLM 服务商,**不是硬性法律障碍**(PDPA 不禁止,中国 PIPL 也可论证可比),但**风险等级显著**(心理数据极敏感 + 中国有政府调取数据法律框架 + AI 子处理者会留存),必须做**传输风险评估 TRIA + ASEAN MCCs 合同 + 数据最小化脱敏 + 用户明示同意**。

5. 🟡 **LLM 跨境连通是核心技术权衡**:**MiniMax 国际站 `minimax.io` 最友好**;智谱国内端点 `bigmodel.cn` 便宜一半但要扛跨境公网、国际端点 `z.ai` 非真正全球且 key 区域隔离;**DeepSeek 最痛**(无海外端点 + 海外慢 30-80s + 注册锁 +86)。阿里云/腾讯云新加坡对中国连通性最好;可选阿里云 Express Connect 专线。**上线前必须 PoC 实测**。

6. 🟢 **危机管道需接新加坡本地真实热线**(已核实 2026 当前号码):**995 救护 / 999 警察 / 1767(SOS 24h)/ 1771(National Mindline,2025-06-18 启用)/ IMH 6389 2222**。

## 2. 医疗边界(HSA / MOH)—— 非医疗陪伴不触发

**判定框架**(来源:[HSA Regulatory Overview](https://www.hsa.gov.sg/medical-devices/regulatory-overview/)、[HSA GL-04 R4 (2025-12)](https://isomer-user-content.by.gov.sg/409/26808a93-6a7a-4551-8c66-13046c6124c5/gl-04-r4-regulatory-guidelines-for-software-medical-devices---a-life-cycle-approach-(2025-dec)-pub.pdf)、[Telehealth GN-13](https://isomer-user-content.by.gov.sg/409/6cd97330-bf07-4251-bb4c-1bc1eaae0edb/regulatory-guidelines-for-telehealth-products-rev-2-1.pdf)):
- "医疗器械"= 用于**诊断/预防/监测/治疗/缓解疾病**且主要靠药理免疫代谢以外手段;SaMD = 用于医疗目的的软件。
- **判定钥匙 = intended purpose(预期用途)+ 是否做医疗声称**,不取决于"它是 App"。
- 即便最低风险 Class A(非无菌)HSA 也**豁免产品注册**。

**本项目结论**:非医疗、不诊断不治疗、情绪陪伴 + 危机引导 → **不是医疗器械,无需 HSA 注册**。

**措辞纪律(合规命门)**:
| 可说 | 绝不可说 |
|---|---|
| 情绪陪伴/倾听/共情、记录心情、低压陪伴、危机时连接专业帮助 | 诊断/治疗/治愈/疗效/临床有效/AI 心理医生/心理疾病筛查诊断 |

**MOH 态度**:鼓励数字心理健康(自建 mindline.sg 国家队),**不审批第三方 App、无准入目录**。([MOH Mental Health Services](https://www.moh.gov.sg/seeking-healthcare/find-a-facility-or-service/mental-health-services/))

## 3. 🔑 数据保护与跨境(PDPA)—— 核心合规

### 3.1 PDPA 主干义务(来源 [PDPC](https://www.pdpc.gov.sg/data-protection-obligations)、[PDPA 全文](https://sso.agc.gov.sg/Act/PDPA2012))
- **适用**:服务器/网站/App 在新加坡、面向新加坡用户 → 落入 PDPA 管辖。
- **DPO 强制**:所有组织须任命数据保护官并**公开联系方式**(可外包)。([PDPC DPO](https://www.pdpc.gov.sg/organisations/resources/getting-started-as-a-data-protection-officer-dpo))
- **数据泄露通知(2021 起)强制**:可通报门槛 = ① 影响 ≥500 人 或 ② 可能造成**重大伤害**(心理健康数据泄露几乎必然满足)。评估期最多 **30 天**,认定后 **3 个日历日内**通报 PDPC,且须通知受影响个人。([Notification of Data Breaches Regs 2021](https://sso.agc.gov.sg/SL/PDPA2012-S64-2021))
- **罚款上限(2022-10-01 提标)**:S$1,000,000 或**新加坡年营业额 10%**,取较高者。([Allen&Gledhill](https://www.allenandgledhill.com/sg/publication/articles/22617/))
- **数据驻留**:**PDPA 无强制本地化**——可存境外,只要满足跨境传输义务。
- **"敏感数据"国情差异**:PDPA **无 GDPR 第 9 条式法定敏感分类**,但健康/心理数据在监管实务中期望**最高级保护**(NRIC 号 2026-12-31 前停用认证;本项目用手机/邮箱无碍)。
- **AI × 个人数据**:2024-03 AI 推荐决策系统指南生效;**2026-06 生成式 AI 指南草案**(咨询至 2026-07-01)——基本为本类应用而写。([PDPC GenAI 公共咨询](https://www.pdpc.gov.sg/organisations/regulations-decisions/regulatory-guidance/public-consultation-on-the-proposed-advisory-guidelines-on-use-of-personal-data-in-generative-ai))

### 3.2 🔴 跨境传输到中国 LLM 深析(本项目核心问题)
数据流:新加坡用户 → 新加坡后端(controller)→ API 发给**中国 LLM 服务商**(智谱/MiniMax/DeepSeek/MiMo,作为境外 processor)。

**(A) PDPA 第 26 条 TLO**:向境外传输须确保接收方提供与 PDPA"可比"保护。新加坡**无充分性白名单**,逐案由传输方举证。实现路径:**合同条款(推荐直接采用 ASEAN 示范合同条款 MCCs)**/BCRs/认证。([PDPC 跨境指南](https://www.pdpc.gov.sg/organisations/resources/guidance-by-topic/guide-to-cross-border-data-transfers)、[ASEAN MCCs](https://www.pdpc.gov.sg/using-asean-mccs-for-cross-border-data-flows-in-singapore))

**(B) TRIA(传输风险评估)—— 真正风险维度**:中国有 PIPL(与 PDPA/GDPR 同源可比 → 有利),但《数据安全法》《国家情报法》《网安法》**存在政府调取数据框架**(新加坡/EU 监管核心关切)。数据是心理类 → 潜在伤害极高 → TRIA 必要性/比例性门槛被推高。

**(C) 中国侧**:数据落地中国即受中国法律管辖;服务商受《生成式 AI 服务管理暂行办法》(2023-08,2025 起实质处罚)、算法备案约束——影响其能如何处理/留存你的数据。([促进和规范数据跨境流动规定 2024-03](https://www.moj.gov.cn/pub/sfbgw/flfggz/flfggzbmgz/202410/t20241030_508738.html))

**(D) 综合判断**:**硬性法律障碍 = 无**;**真实障碍 = 合规工程量 + 风险等级高**(极敏感 + 政府调取框架 + AI 留存三者叠加)。

**(E) 合规动作方向**:
1. **签可比保护合同**(ASEAN MCCs 式 DPA),强制写入:仅为本项目目的处理 / **不得用于服务商模型训练** / 处理完成即删不留日志原文 / 不得向第三方再传输 / 政府调取时通知你。
2. **做并留存 TRIA**(每家服务商:必要性/目的地法律/安全措施/补救)。
3. **数据最小化/脱敏(最重要工程防线)**:发 LLM 前 prompt 脱敏(剔除直接标识);评估新加坡/东盟部署或本地化推理替代直连 `open.bigmodel.cn`。
4. **用户告知与明示同意**:隐私政策明确披露传输到中国 + 具体服务商 + 目的 + 中国保护水平 + 风险;注册/首次对话取得**单独明示同意**。
5. **服务商尽调**(ISO27001/SOC2、是否完成生成式 AI 备案)。
6. **降级方案**:把"LLM 提供商"做成可替换(新加坡/东盟/欧美模型,各走一次跨境评估)。

## 7. 🔑 LLM 跨境连通(核心技术权衡)

> 部署在新加坡的第一技术约束。各服务商端点真相(来源见下):

| 服务商 | 端点 | 新加坡访问友好度 | 备注 |
|---|---|---|---|
| **MiniMax** | 国际 `minimax.io` / 国内 `minimaxi.com` | **国际端点最友好** ⭐ | key 区域隔离;优先用 `minimax.io`([platform.minimax.io](https://platform.minimax.io/)) |
| 智谱 GLM | 国内 `open.bigmodel.cn` / 国际 `open.z.ai` | 国内便宜约 50% 但扛跨境公网;z.ai **非真正全球部署**且 key 区域隔离 | 本项目用 glm-4.7 走 `bigmodel.cn`([Cline Z AI](https://docs.cline.bot/provider-config/zai)) |
| **DeepSeek** | `api.deepseek.com`(单一,中国境内) | **最痛**:海外 30-80s、注册锁 +86、无海外端点 | 建议降级备份或第三方托管(注意合规)([DeepSeek docs](https://api-docs.deepseek.com/)) |

**新加坡→中国公网**:典型延迟 ~80-130ms(优于美欧,略逊港日),长 SSE 跨境有抖动/丢包/偶发重置 —— Aurora 流式回复、主动对话会受影响。**数字多来自社区/供应商,上线前必须 PoC 实测。**

**稳定跨境选项**:
- **阿里云 Express Connect / CEN**(新加坡↔中国多城市专线,支持三大运营商)。([Express Connect](https://www.alibabacloud.com/en/product/express-connect))
- 在新加坡后端与国内 LLM 间加 **LLM Gateway**(熔断/降级/缓存/多端点路由);本项目 Spring Boot 已有 mode/temperature 抽象可承载。
- 量级上来可在香港设 LLM 代理跳板(港→中骨干最优、港→新加坡近)。

## 8. 商业 / 税务 / 支付

### 现阶段(个人/中国实体直接发行)—— 推荐
- 以中国个人/实体注册 Apple($99/yr)/Google($25)开发者账号,发行到新加坡**完全可行**,无需新加坡实体。([Apple D-U-N-S](https://developer.apple.com/help/account/membership/D-U-N-S/))
- **GST 与你无关**:走 Apple/Google 内购,GST 由平台代收代缴(9%)。自建支付仅当**全球 >S$1M 且新加坡 B2C >S$10 万**双触发才需 OVR 自注册——远未到的阶段。([IRAS OVR](https://www.iras.gov.sg/taxes/goods-services-tax-(gst)/gst-and-digital-economy/overseas-businesses))
- **支付牌照不需要**:卖数字订阅不是受监管支付服务;用 Stripe/商店内购(牌照在它们手上)。([PSA 2019](https://sso.agc.gov.sg/act/psa2019))

### 注册新加坡 Pte.Ltd.(仅商业化做大/拿补贴/品牌实体时)
- **硬门槛 = 必须一名本地常住董事**(公民/PR/EntrePass/EP),外国人通常靠**挂名董事 S$2,000–4,000/年**解决。注册费 S$315,全程可远程。([ACRA](https://www.acra.gov.sg/register/foreign-business/ways-to-set-up/))
- 银行:Aspire/Wise Business/Airwallex 远程开户(3 天),非传统三大行高门槛。([Intraconnect](https://intraconnect.biz/open-a-business-bank-account-in-singapore-as-a-foreigner/))
- **`.com.sg` 需 ACRA 公司**;版权新加坡自动保护无登记(中国建议做软著登记)。

## 14. 一句话总评
**新加坡是本项目最现实的海外发行地**——相比中国大陆,它**免掉了备案/软著/多商店/境内主体**这整面行政墙;相比纯海外,它**法规清晰、AI 治理友好、华语市场贴近**。现阶段以中国个人身份直接上架、GST 由平台代收,几乎零成本即可验证。真正需要认真对待的只有两件事:**(1) PDPA 跨境传输到中国 LLM 的合规链**(TRIA + ASEAN MCCs 合同 + 数据最小化 + 明示同意);**(2) 新加坡↔中国 LLM 的网络连通**(MiniMax 国际站最稳,智谱扛跨境,DeepSeek 最痛,上线前必 PoC)。商业化做大或要拿补贴时,再注册新加坡公司走 Startup SG Tech 窄路。
